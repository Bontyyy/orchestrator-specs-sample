# FEAT-2026-0006 — Widget count endpoint

## Feature summary

Expose a lightweight endpoint that returns the total number of widgets currently persisted, without returning the widget payloads themselves. Intended for admin dashboard display of the aggregate count without the cost of paging the full list.

## Scope

- Count all widgets regardless of status or any other attribute.

## Out of scope

- Filtering by status, owner, creation date, or any other criterion.
- Caching of the count across requests.
- Returning any widget fields beyond the count.

## Acceptance criteria

- **AC-1** — `GET /widgets/count` returns HTTP 200 with a JSON response body `{"count": N}` where `N` is a non-negative integer equal to the total number of widgets currently persisted.

## Related specs

Narrative-form spec only at v1 (no OpenAPI file); the `## Acceptance criteria` section above is the authoritative behavior contract that qa-authoring enumerates against.
