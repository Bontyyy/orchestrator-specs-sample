# FEAT-2026-0009 — Bulk widget creation endpoint

## Feature summary

Adds a bulk-create endpoint `POST /widgets/bulk` to the existing widget API so that clients can create up to 50 widget objects in a single request. Validation is atomic: if any widget in the batch fails validation, none are persisted.

## Scope

- A single new endpoint `POST /widgets/bulk` on the existing `WidgetsController`.
- The request body accepts a JSON array of widget objects; each object must supply `name` (string), `sku` (string), and `quantity` (integer).
- The maximum batch size is 50 items. Requests with more than 50 items are rejected before any validation or persistence occurs.
- Validation is atomic — if any widget in the batch fails field-level validation, the entire batch is rejected and zero widgets are persisted.
- On success, the endpoint returns HTTP 201 with a JSON array of created widget objects, each including a generated `id`.
- On batch-size overflow (more than 50 items), the endpoint returns HTTP 400 with `error.code = "batch_size_exceeded"` and an `error.message` describing the 50-item limit.
- On any validation failure, the endpoint returns HTTP 422 with `error.code = "batch_validation_failure"` and an `error.failures` array. Each entry in `error.failures` has an `index` (0-based position of the failing item in the input array) and a `reason` (human-readable validation message describing why that item failed).
- Field-level validation reuses the existing widget rules: `name` non-blank, `sku` non-blank, `quantity` in the range 0–10000.

## Out of scope

- Bulk update and bulk delete operations.
- Streaming or asynchronous batch processing.
- Idempotency keys or deduplication.
- Partial success responses (the endpoint is all-or-nothing; no 207 Multi-Status).

## Acceptance criteria

- **AC-1** — `POST /widgets/bulk` with a JSON array of 1–50 valid widget objects (each with non-blank `name`, non-blank `sku`, and `quantity` in 0–10000) returns HTTP 201 with a JSON array of created widget objects where each element includes a generated `id` field alongside the submitted `name`, `sku`, and `quantity`.

- **AC-2** — `POST /widgets/bulk` with a JSON array containing more than 50 items returns HTTP 400 with a JSON response body where `error.code` equals the literal string `"batch_size_exceeded"` and `error.message` is a non-empty string describing the 50-item limit.

- **AC-3** — `POST /widgets/bulk` where any widget in the batch fails field-level validation (blank `name`, blank `sku`, or `quantity` outside 0–10000) returns HTTP 422 with a JSON response body where `error.code` equals the literal string `"batch_validation_failure"` and `error.failures` is a non-empty array; each entry in the array has an `index` field (0-based integer matching the position of the failing item in the input array) and a `reason` field (non-empty string). Zero widgets are persisted — the full batch is rejected.
