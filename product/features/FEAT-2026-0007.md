# FEAT-2026-0007 — Widgets list with pagination

## Feature summary

Paginate the `GET /widgets` listing endpoint so that a client can request a bounded slice of the widget corpus in a single response, with explicit rejection of over-large slice requests.

## Scope

- Default slice of up to 50 widgets when `page_size` is absent.
- Explicit slice of up to N widgets for `1 ≤ N ≤ 500` via `page_size=N`.
- Structured rejection of `page_size > 500` with HTTP 400 and error code `page_size_over_limit`.

## Out of scope

- Cursor- or offset-based navigation beyond `page_size`.
- Sorting options.
- Filtering by any field.
- Behavior for non-integer or negative `page_size` values (handled by existing request-validation middleware).

## Acceptance criteria

- **AC-1** — `GET /widgets` with no `page_size` query parameter returns HTTP 200 with a JSON array of up to 50 widget objects.
- **AC-2** — `GET /widgets?page_size=N` with `1 ≤ N ≤ 500` returns HTTP 200 with a JSON array of up to N widget objects.
- **AC-3** — `GET /widgets?page_size=N` with `N > 500` returns HTTP 400 with a JSON response body shaped as `{"error": {"code": "page_size_over_limit", "message": "..."}}`. The `error.code` field must equal the literal string `page_size_over_limit`.
