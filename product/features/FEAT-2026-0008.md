# FEAT-2026-0008 — Widget update endpoint

## Feature summary

Adds a PATCH `/widgets/:id` endpoint to the existing widget API so that clients can partially update a widget's fields. Only the fields provided in the request body are updated; omitted fields retain their current values.

## Scope

- A single new endpoint `PATCH /widgets/:id` on the existing `WidgetsController`.
- The request body accepts any non-empty subset of `{name, sku, quantity}`. Omitted fields are left unchanged.
- On success, the endpoint returns HTTP 200 with the full updated widget JSON.
- On a non-existent `:id`, the endpoint returns HTTP 404 with a JSON error body containing `error.code = "widget_not_found"`.
- Field-level validation reuses the existing `CreateWidget` rules: `name` non-blank, `sku` non-blank, `quantity` in the range 0–10000.
- Uses the existing `Widget` model and `InMemoryWidgetRepository` without modification.

## Out of scope

- Full replacement semantics (PUT).
- Bulk partial updates.
- ETag or concurrency-control headers.
- Field-level validation rules beyond those already enforced by `CreateWidget` (name non-blank, sku non-blank, quantity 0–10000).

## Acceptance criteria

- **AC-1** — `PATCH /widgets/:id` with a valid partial body (any non-empty subset of `{name, sku, quantity}`) returns HTTP 200 with the full updated widget JSON. Fields included in the request body reflect their new values; fields omitted from the request body retain their prior values unchanged.

- **AC-2** — `PATCH /widgets/:id` where `:id` does not correspond to any existing widget returns HTTP 404 with a JSON response body where `error.code` equals the literal string `"widget_not_found"`.
